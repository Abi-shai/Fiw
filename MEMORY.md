## Contexte actuel

Sitemaps Fiw (Client) et Fiw Pro (Prestataire) complétés. ADR 0003 décidé : AffiliéPartenaire dans Fiw Pro uniquement. **Deux prototypes Expo opérationnels** : `apps/fiw/` (flux Transport Client — `home → configure → searching → course-active → cloture`) et `apps/fiw-pro/` (flux Mission active Transport Prestataire, complet). Architecture : deux projets Expo indépendants. Style guide validé : couleur primaire Fiw `#0066FF` (bleu), tokens dans `apps/fiw/constants/`. Prochain jalon : tests utilisateurs terrain sur le flux Transport Client (`apps/fiw/`) — priorité frais de rapprochement (Options A/B/C).

**Flux Livraison client v1 (12 juillet 2026)** : benchmark Mobbin ([docs/benchmark-livraison-mobbin.md](docs/benchmark-livraison-mobbin.md)) puis prototype complet `apps/fiw/app/livraison/` — `configure` (feuille 3 crans ; type/taille de colis en chips ; destinataire **avec** le colis, choisi d'abord dans les contacts façon Careem) → `options` (feuille 3 crans ; gammes Vélo Express / Moto Livraison + paiement, **sans « Quand ? » ni Total** — départ immédiat, le prix définitif se joue en mise en relation ; `WheelPicker` construit puis descopé avec la planifiée, reste au DS pour Yobanté) → `searching` (**hub 4 issues** : near / far frais / **groupage détecté** / none — la **livraison groupée est une proposition de l'algorithme** en Option A/B « delta héros », attente du seuil 2 commandes avec dénouement confirmé/échec → prix normal, aligné Product Doc « Détection automatique ») → `suivi` (feuille 3 crans, jalons `StepProgress`, **code de remise** `CodePill`, annulation gratuite avant collecte seulement) → `cloture` (reçu + avis). Nouveaux composants : `ChipGroup`, `StepProgress`, `CodePill`, `WheelPicker`, `PaymentSheet` (extrait de configure Transport), `VehicleBlock`/`RouteCard` étendus. Vélo Express **sans illustration isométrique** (icône `bicycle` en attendant l'asset). Yobanté hors périmètre v1 (patterns documentés dans le benchmark).

**Dernière itération Transport (juillet 2026)** : frais d'attente annoncés dès la commande (5 min offertes puis 100 F/min — cf. CONTEXT.md), compte à rebours d'arrivée réel, **Option C** présentée via un interrupteur de démo sur `searching` (A/B et C exclusifs), écran **Historique** (`app/history/`) avec plaque mais **sans numéro prestataire** (objet oublié → service client), actions course active câblées en retour léger réel (appel masqué `transport/call`, chat `transport/chat`, SOS, partage), composants partagés `ReceiptCard` + `ScreenHeader` ajoutés, tokens couleur étendus (`track`/`surfaceAlt`/`brandYellow`) + variante typo `displayXl`. Écran mort `transport/destination` supprimé.
